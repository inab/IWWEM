IWWE&M v0.7.0 workflow versioning scheme

One of the main needs on future workflow development environments will be a workflow versioning mechanism, currently unsupported by tools like Taverna. As IWWE&M will host in the near future workflows for different workflow engines, this mechanism should be integrated into the workflow repository itself, as it is going to be taken as the reference point for those hosted workflows.

The workflow versioning mechanism used by IWWE&M is based on de facto software versioning style. The system is based on a 3 numbers mechanism, separated by dots, and the change of those numbers is strictly managed by IWWE&M. This is the meaning of each one, with an example of version 2.1.3:

2.

  • The workflow has been updated in an incompatible way: some input or output has disappeared or changed its name.
  • Workflow responsible has requested this major version.

1.

  • MIME types associated to some input or output have been changed
  • The workflow distribution license has been changed
  • The workflow responsible, which is an IWWE&M concept, has changed (different from workflow author)
  • The workflow has been updated in an compatible way: the workflow keeps at least the same inputs and outputs.

3

  • There has been one or more changes in workflow input or output descriptions.
  • The workflow author has changed (different from workflow responsible, an IWWE&M concept)

IWWE&M allows creating a new version of the same or upper class for a workflow when it is the newest one from its branch. For instance, if workflow A has versions 1.0.0 and 1.0.1, then you cannot create a new version of the workflow version 1.0.0 unless it is a major compatible change, so it would be 1.1.0, or a major incompatible change, so it would be 2.0.0. The responsible is only capable to generate a 1.0.2 version from 1.0.1 one.

Also, each workflow version's reponsible can consider that the workflow life cycle has finished, so IWWE&M has considered a new workflow flag called deprecated, which warns the potential users about its lack of usability (perhaps due the disappear of a service used by the workflow).
